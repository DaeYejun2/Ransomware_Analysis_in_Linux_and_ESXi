1. 가상화 인프라 침투 및 제어 매커니즘 (RQ1)
    * 랜섬웨어가 일반적인 리눅스 환경과 달리 ESXi 환경에서 어떻게 Hypervisor에 특화된 동작을 수행하는지 분석했다.
        1. 정교한 API 활용: 분석 결과, Blackmatter와 Darkside 같은 고급 패밀리는 단순 셸 명령어를 넘어 pyvmomi(vSphere SDK)를 직접 호출한다. 이를 통해 관리자 세션을 탈취하고, 하이퍼바이저 레벨에서 가상 머신의 전원을 제어하거나 스냅샷을 삭제하는 행위를 보였다.
        2. 관리 서비스 무력화: 샘플의 대다수가 실행 직후 hostd(호스트 관리) 및 vpxa(vCenter 에이전트) 서비스를 강제 종료한다. 이는 VMDK 파일에 걸린 파일 시스템 Lock을 해제하여 암호화를 원활하게 하고, 관리자의 접근을 차단하는 이중 목적을 가진다.
        3. 하드코딩된 경로 탐색: 샘플들은 /vmfs/volumes/와 같은 ESXi 전용 스토리지 경로를 최우선적으로 탐색하여, 일반 시스템 파일이 아닌 가상 디스크 파일을 정밀 타격합니다. 이 부분이 서비스가 실제로 돌아가는 가상 머신의 디스크 파일이기 때문. 이 파일 안에서 경로 파악 -> 대상 식별 -> 프로세스 킬 -> 암호화로 이어지는 킬 체인의 시작점이 된다.
     
2. 대용량 VMDK 암호화 전략의 효율성 (RQ2)
   * 수십 GB에 달하는 가상 디스크를 단시간에 무력화하기 위해 랜섬웨어가 사용하는 최적화 기법을 정량적으로 분석했다.
     1. 간헐적 암호화: 샘플의 85%가 파일 전체 암호화 대신 부분 암호화 사용
     2. 파일 크기에 상관없이 범위를 제한하는 설정값으로 암호화 ([CFG] Part Size: 300mb)
     3. 데이터 전체보다 핵심 파일 헤더와 메타데이터 파괴를 우선시하여, 수 초 만에 인프라 전체를 사용 불능 상태로 만듦
    
3. 인프라 고립 및 탐지 회피 (RQ3)
   * 공격자가 Hypervisor를 외부로부터 고립시키고 분석을 방해하기 위해 수행하는 고도화된 전략 관찰.
       1. BlackMatter 등은 SSH, NFS, SNMP 등의 서비스에 대한 방화벽 규칙을 수십 회 반복 수정하여, 외부 관리 서버나 백업 서버가 하이퍼바이저에 접속하지 못하도록 물리적으로 고립시킨다.
       2. REvil 패밀리 분석 중 가장 특이한 현상으로, 공격자가 SSL 인증서를 무효화하고 핵심 관리 프로세스를 파괴함에 따라 vCenter에서 해당 호스트의 CPU 사용량이 **NoneMHz**로 표시되는 현상을 발견했다.
          이는 실제 암호화가 0초 만에 끝난 것이 아니라, 보안 모니터링 도구를 먼저 차단하여 공격 행위가 로그에 기록되지 않도록 유도한 결과임을 확인했다.
